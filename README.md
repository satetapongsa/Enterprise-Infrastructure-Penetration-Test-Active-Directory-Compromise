# 🛡️ Enterprise Network Penetration Test & Active Directory Compromise

**Difficulty Level:** Advanced  
**Focus Areas:** Network Pivoting, Active Directory Exploitation, Lateral Movement, Cisco Routing Misconfigurations

## 📋 Project Overview
This repository contains a comprehensive penetration testing write-up for a simulated enterprise network environment. The objective was to simulate a real-world external breach, navigate through segmented internal networks, and ultimately achieve a complete domain takeover.

The target infrastructure consists of three primary departments, segmented by Cisco networking equipment:
1. **SERVICE (DMZ):** Public-facing web servers and helpdesk systems.
2. **ACCOUNT (Internal):** File shares, databases, and employee workstations.
3. **IT (Core Infrastructure):** Active Directory Domain Controllers and centralized management servers.

## ⚔️ Attack Kill Chain

### 1. Initial Access (SERVICE Network)
* **Target:** Helpdesk Ticketing System (Port 8080)
* **Vulnerability:** Insecure Deserialization (Java-based application).
* **Action:** Exploited using a custom payload generated by `ysoserial` to achieve a reverse shell.
* **Result:** `service_svc` user access on the web server within the `10.10.10.0/24` subnet.

### 2. Network Reconnaissance & Pivoting
* **Discovery:** Analyzed the routing table and discovered active routes to the ACCOUNT and IT subnets. 
* **Action:** Deployed `Chisel` to create a SOCKS5 reverse proxy, establishing a tunnel from the external attacker machine directly into the internal network, bypassing edge firewalls.

### 3. Lateral Movement (ACCOUNT Network)
* **Target:** Internal File Share (SMB) in the `10.10.20.0/24` subnet.
* **Vulnerability:** Lack of SMB Signing.
* **Action:** Performed LLMNR/NBT-NS Poisoning using `Responder`. Captured and relayed NTLM authentication hashes to the SMB server using `ntlmrelayx.py`.
* **Result:** Gained Local Administrator execution on the accounting server. Extracted credentials from memory using `Mimikatz`.

### 4. Domain Privilege Escalation (IT Network)
* **Target:** Domain Controller (`10.10.30.5`)
* **Vulnerability:** Active Directory Certificate Services (AD CS) ESC8 (NTLM Relay to Web Enrollment).
* **Action:** Coerced the Domain Controller to authenticate using `PetitPotam`. Relayed the machine account hash to the AD CS HTTP endpoint to request a client certificate.
* **Result:** Requested a TGT using the forged certificate, executed a DCSync attack, and dumped the `NTDS.dit` hashes. Achieved **Domain Admin** compromise.

## 🛡️ Remediation & Hardening Recommendations

To secure the infrastructure against this attack path, the following structural and system-level changes are required:

### 1. Network Segmentation & Cisco ACL Hardening
* Implement strict Access Control Lists (ACLs) on the Cisco core switches.
* The SERVICE VLAN (DMZ) must not have unrestricted routing access to the ACCOUNT or IT VLANs. Implement a default-deny policy and allow only required ports (e.g., specific database ports).

### 2. Prevent NTLM Relay Attacks
* **Enforce SMB Signing:** Apply a Global Group Policy (GPO) to require SMB signing across all workstations and servers.
* **Disable LLMNR and NBT-NS:** Prevent local network poisoning by disabling these legacy protocols via GPO.

### 3. Active Directory Certificate Services (AD CS) Hardening
* Disable NTLM authentication on the AD CS Web Enrollment interface and enforce Kerberos, or disable the HTTP endpoint entirely (require HTTPS / EPA).
* Regularly audit certificate templates for misconfigurations (ESC1-ESC8).
